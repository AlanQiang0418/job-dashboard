# 求职申请管理看板

当前版本：`v1.5.12`

一个面向大学生求职季场景的静态网页原型，用于集中管理申请进度、截止日期、材料准备、面试安排与后续行动。项目强调产品结构表达和演示完整度，适合作为产品设计测评题的作品提交。

## 项目目标

这个原型主要解决 4 类典型问题：

- 申请信息分散：公司、岗位、状态、下一步动作分散在聊天记录、备忘录和表格中，不便统一管理
- 截止日期容易遗漏：多个岗位同时推进时，最紧急事项不够清晰
- 材料版本混乱：不同岗位需要不同简历、作品集、成绩单版本，复用成本高
- 面试推进难跟踪：从待投递到 Offer/结果，缺少一个可视化的全流程面板

对应的产品设计目标是：

- 用一个页面管理整个求职流程
- 同时提供“总览视图”和“精细管理视图”
- 让用户快速知道当前该优先处理什么
- 在静态部署环境下也能稳定演示完整交互

## 主要功能

### 1. 首页总览

首页顶部包含产品标题、核心说明和几个主操作入口：

- `新增申请`
- `导出周报`
- `导出到日历`
- `Reset`

右上角保留“当前急需处理”卡片，用于突出最值得优先处理的岗位。

### 2. 指标总览

页面顶部有 4 个核心指标卡：

- 投递总数
- 本周截止
- 面试推进率
- 材料完整率

其中前 3 个指标支持点击跳转到对应的筛选结果：

- 点击“投递总数”跳到表格管理下的“全部”
- 点击“本周截止”跳到表格管理下的“本周截止”
- 点击“面试推进率”跳到表格管理下的“笔试/面试”

### 3. 任务中心

任务中心只展示当前真正需要处理的任务，而不是简单罗列所有申请。

当前规则聚焦以下两类：

- `待完善材料`
- `待决定`

任务会按轻重缓急自动排序，便于用户先处理高优先级、临近截止的事项。

### 4. 申请进度看板

看板按 4 个阶段管理岗位：

- 待投递
- 已投递
- 笔试/面试
- Offer/结果

每张卡片展示：

- 公司
- 岗位
- 当前状态
- 优先级
- 城市
- 截止日期
- 下一步动作

桌面端：

- 使用 4 列看板布局
- 适合做整体进度总览

移动端：

- 改为横向滑动查看不同阶段
- 页面内有明确提示“左右滑动查看不同申请阶段”
- 当筛选或搜索后只剩一个阶段时，看板会切换为单栏并锁定横向滑动

### 5. 表格管理视图

表格视图适合做更细的筛选和排序，支持：

- 搜索公司 / 岗位 / 地区
- 阶段筛选
- 本周截止筛选
- `Due Date` 排序
- `进度` 排序
- `优先级` 排序

看板视图与表格视图共享同一套搜索词和筛选条件。

### 6. 详情抽屉

点击看板卡片、任务中心、面试日程或表格行，可以打开右侧详情抽屉。

抽屉中可以查看：

- 投递渠道
- 联系人
- 岗位链接
- 当前阶段
- 当前状态
- 优先级
- 下一步动作
- 备注

进入编辑模式后，可以修改：

- 申请阶段
- 截止日期
- 当前状态
- 优先级
- 联系人
- 岗位链接
- 下一步动作
- 备注

同时支持删除岗位，并带二次确认弹窗。

### 7. 新增申请

新增申请弹窗目前仅要求以下字段必填：

- 公司
- 岗位
- 申请阶段
- 当前状态

其余字段均可选：

- 城市
- 截止日期
- 联系人
- 岗位链接
- 下一步动作
- 备注

这样可以兼顾“快速录入”和“后续补全”两种使用习惯。

### 8. 面试日程

面试日程模块用于展示近期的面试和笔试安排，并支持从日程项直接查看对应申请详情。

### 9. 推荐岗位

推荐岗位模块提供轻量推荐能力，用于补充机会池。

支持：

- 刷新推荐
- 将推荐岗位加入待投递

### 10. 材料中心

材料中心统一展示常用求职材料，并提供示例导出：

- 中文简历：导出示例 `.pdf`
- 英文简历：导出示例 `.pdf`
- 作品集：导出示例 `.zip`
- 成绩单 / 证明：导出示例 `.zip`

这里的导出文件是演示用空示例文件，目的是保证静态部署时行为稳定、可展示。

### 11. 导出功能

当前提供两种导出：

- `导出周报`
  - 导出 `.csv`
  - 更接近真实管理工具的数据导出方式

- `导出到日历`
  - 当前导出示例 `.ics`
  - 用于演示“导出到系统日历”的交互链路
  - 在 GitHub Pages 这类纯静态环境中更稳定

### 12. 本地持久化

项目接入了浏览器 `localStorage`：

- 新增、编辑、删除后会自动保存
- 刷新页面后数据不会丢失
- 点击 `Reset` 可以恢复默认示例数据

说明：

- 本地打开页面可用
- 部署到 GitHub Pages 后同样可用
- 仅保存在当前浏览器、当前设备，不会跨设备同步

## 使用方式

### 本地预览

直接打开 [index.html](./index.html) 即可预览。

### 静态部署

本项目不依赖后端服务，可以直接部署到任意静态托管平台，例如：

- GitHub Pages
- Netlify
- Vercel

## 文件结构

- `index.html`
  - 页面结构
  - 看板、表格、弹窗、抽屉、模板

- `styles.css`
  - 页面视觉样式
  - 响应式布局
  - 移动端横向滑动看板适配

- `script.js`
  - 示例数据
  - 看板 / 表格渲染
  - 搜索、筛选、排序
  - 抽屉编辑
  - 导出逻辑
  - `localStorage` 持久化

## 移动端说明

移动端与桌面端的交互策略不同：

- 桌面端更强调多列并行浏览
- 移动端更强调阶段切换和内容可读性

因此当前移动端采用：

- 横向滑动阶段列
- 搜索 / 筛选后自动隐藏无结果阶段
- 单阶段结果下禁止多余横向滑动

这是为了避免在窄屏里硬塞 4 列导致信息不可读。

## 当前实现边界

这个项目是前端静态原型，不包含真实后端能力，因此以下内容是演示级实现：

- 不接真实招聘平台 API
- 不做账号系统
- 不做多人协作
- 不做跨设备同步
- 材料导出与日历导出为示例文件

这些取舍是为了保证作品在测评场景下：

- 稳定
- 易部署
- 易演示
- 不依赖额外服务

## 适合作为提交作品的原因

这个版本的重点不在“把所有技术都做深”，而在于完整表达产品思路：

- 有明确的用户场景
- 有完整的信息结构
- 有从总览到明细的管理链路
- 有桌面端和移动端两套交互策略
- 能以链接形式直接提交和演示

如果后续继续迭代，优先方向会是：

- 账号体系与云端同步
- 真正的日历写入能力
- 与招聘网站或邮箱的真实联动
- 更细粒度的数据统计和复盘分析
