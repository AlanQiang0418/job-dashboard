# 求职申请管理看板

这是一个面向大学生求职季场景的单页网页原型，重点解决“申请信息分散、截止日期容易遗漏、材料版本混乱、面试推进难跟踪”这几个核心问题。项目适合作为产品设计测评题的演示作品，也可以直接作为静态作品集页面展示。

## 项目定位

这个原型不是偏技术实现的招聘系统，而是偏产品结构表达的管理工具。核心目标是帮助用户把求职过程从“零散记录”变成“可追踪、可筛选、可行动”的单一工作台。

页面围绕以下使用场景设计：

- 同时管理多个公司的申请状态
- 快速查看哪些岗位最需要优先处理
- 统一管理简历、作品集、成绩单等投递材料
- 跟进面试安排、Offer 结果和后续动作
- 在看板视图和表格视图之间切换，分别满足总览和精细管理

## 功能概览

### 1. 首页总览

- 顶部展示产品简介和核心操作入口
- 支持新增申请、导出周报、导出示例日历文件、重置示例数据
- 右侧展示“当前急需处理”卡片，突出最需要关注的申请

### 2. 核心指标

- 投递总数
- 本周截止
- 面试推进率
- 材料完整率

其中前三个指标支持点击联动到表格管理视图，对应筛选条件会自动切换，方便从概览快速进入明细管理。

### 3. 任务中心

- 仅展示当前需要处理的关键任务
- 当前规则聚焦 `待完善材料` 和 `待决定`
- 按轻重缓急排序，帮助用户优先处理高优先级岗位

### 4. 申请进度看板

看板分为四个阶段：

- 待投递
- 已投递
- 笔试/面试
- Offer/结果

每张卡片包含：

- 公司与岗位
- 当前状态
- 优先级
- 城市
- 截止日期
- 下一步动作

桌面端保留看板式分列管理，移动端则切换为单列分组列表，以保证信息可读性。

### 5. 表格管理视图

表格视图适合做精细筛选和逐项管理，支持：

- 搜索公司 / 岗位 / 地区
- 阶段筛选
- 本周截止筛选
- 按 Due Date、进度、优先级排序

### 6. 详情抽屉

点击任务中心、看板卡片、时间线或表格行，都可以打开右侧详情抽屉。

抽屉内支持：

- 查看投递渠道、联系人、岗位链接
- 查看当前阶段、当前状态、优先级、下一步动作、备注
- 进入编辑模式修改申请信息
- 删除岗位

### 7. 材料中心

材料中心用于统一展示常用投递材料，并提供示例导出：

- 中文简历导出 `.pdf`
- 英文简介导出 `.pdf`
- 作品集和成绩单导出 `.zip`

当前导出为示例文件，主要用于作品演示。

### 8. 示例日历导出

“导出到日历”当前导出的是一个示例 `.ics` 文件，用于演示交互流程。这样在 GitHub Pages 这类纯静态部署环境下也能稳定工作。

### 9. 本地持久化

项目已接入浏览器 `localStorage`：

- 新增、编辑、删除岗位后会自动保存
- 刷新页面后数据不会丢失
- 支持通过 `Reset` 按钮恢复默认示例数据

这项能力在本地打开页面时可用，在 GitHub Pages 部署后同样可用，但只会保存在当前浏览器和当前设备内，不会跨设备同步。

## 使用方式

直接打开 `index.html` 即可预览。

如果部署到 GitHub Pages、Netlify、Vercel 等静态托管平台，也不需要后端服务器即可运行。

## 文件说明

- `index.html`：页面结构、弹窗、抽屉、模板
- `styles.css`：整体视觉样式、响应式布局、移动端适配
- `script.js`：示例数据、筛选排序、看板渲染、详情编辑、导出逻辑、本地持久化

## 当前实现边界

这是一个前端静态原型，因此以下能力目前是演示级实现：

- 不接真实后端
- 不做账号体系
- 不做跨设备同步
- 材料导出和日历导出为示例文件

这些选择是为了保证作品在测评提交场景中足够稳定、可展示、可部署。
