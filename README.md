# 求职申请管理看板

当前版本：`v1.7`

一个面向大学生求职季的静态网页原型，用于集中管理求职申请、截止日期、材料状态、笔试/面试安排和后续行动。项目不依赖后端服务，适合直接本地预览或部署到 GitHub Pages 等静态托管平台。

## 核心目标

这个产品主要解决求职季中 4 类高频问题：

- 申请信息分散：公司、岗位、渠道、联系人、备注和下一步动作缺少统一入口。
- 截止日期容易遗漏：多个岗位同时推进时，很难判断当前最该处理哪一个。
- 面试推进难跟踪：从待投递、已投递、笔试/面试到 Offer/结果，缺少连续的进度视图。
- 材料复用成本高：简历、作品集、成绩单等材料需要统一管理和快速导出。

设计目标是让用户在一个页面里完成“总览、筛选、编辑、复盘和导出”，并能清楚知道下一步该做什么。

## 快速使用

### 本地预览

直接打开 [index.html](./index.html) 即可预览。

### 静态部署

项目是纯前端静态页面，不需要服务器。可以直接部署到：

- GitHub Pages
- Netlify
- Vercel
- 其他静态托管平台

### 本地数据

项目使用浏览器 `localStorage` 持久化数据：

- 新增、编辑、删除申请后会自动保存。
- 刷新页面后申请记录不会丢失。
- 点击 `RESET` 可以恢复默认示例数据。
- 数据仅保存在当前浏览器和当前设备，不会跨设备同步。

## 主要功能

### 1. 首页总览

首页提供全局操作入口和关键指标：

- `新增申请`：快速录入公司、岗位、阶段、当前状态等信息。
- `导出周报`：导出 `.csv`，便于进一步整理或复盘。
- `导出到日历`：下载示例 `.ics` 文件，模拟同步到系统日历的链路。
- `README`：打开项目说明文档。
- `RESET`：恢复默认示例数据。

顶部指标包含：

- 投递总数
- 本周截止
- 面试推进率
- 材料完整率

其中投递总数、本周截止、面试推进率可以点击，并自动跳转到表格视图中对应的筛选结果。

### 2. 任务中心

任务中心只展示真正需要处理的任务，而不是罗列所有申请。

当前重点处理两类状态：

- `待完善材料`
- `待决定`

任务会按优先级和截止日期排序，便于先处理高风险、高优先级事项。

### 3. 申请视图

项目提供三种互补视图，使用同一套申请数据：

- 看板视图：按阶段展示，适合总览整体进度。
- 表格视图：适合搜索、筛选、排序和批量查看。
- 日历视图：按周展示 due date、笔试和面试安排。

看板阶段包括：

- 待投递
- 已投递
- 笔试/面试
- Offer/结果

表格视图支持：

- 搜索公司 / 岗位 / 地区
- 阶段筛选
- 本周截止筛选
- `Due Date` 排序
- `进度` 排序
- `优先级` 排序

### 4. 日历与排期

日历视图以周为单位展示每日任务：

- 全天区域显示各岗位截止日期。
- 时间网格显示笔试和面试安排。
- 支持 `Today`、上一周、下一周切换。
- 支持和看板 / 表格一致的搜索与筛选。
- 点击任意日历任务可打开对应申请详情。

新增申请或编辑申请时，如果满足以下条件，会出现可选的排期字段：

- 申请阶段为 `笔试/面试`
- 当前状态为 `待笔试`、`待一面`、`待二面`、`待三面` 或 `待HR面`

排期输入由两部分组成：

- 日期：使用和截止日期一致的日历选择器。
- 时间：使用固定下拉列表选择。

如果当前状态还没有正式 schedule，可以留空，不影响保存申请。

### 5. 申请详情抽屉

点击看板卡片、任务中心、表格行、推荐岗位或日历任务，都会打开同一个详情抽屉。

抽屉中可以查看：

- 投递渠道
- 联系人
- 岗位链接
- 当前阶段
- 当前状态
- 优先级
- 笔试 / 面试日期和时间
- 下一步动作
- 备注

进入编辑模式后可以修改：

- 申请阶段
- 当前状态
- 截止日期
- 优先级
- 笔试 / 面试日期和时间
- 联系人
- 岗位链接
- 下一步动作
- 备注

删除岗位时会出现二次确认弹窗，避免误删。

### 6. 推荐岗位

推荐岗位是轻量机会池功能，用于补充待投递来源。

支持：

- 刷新推荐
- 将推荐岗位加入待投递
- 加入后继续在看板、表格和详情抽屉中管理

### 7. 材料中心

材料中心把常用求职材料作为可行动入口展示，并提供示例导出：

- 中文简历：导出示例 `.pdf`
- 英文简历：导出示例 `.pdf`
- 作品集：导出示例 `.zip`
- 成绩单 / 证明：导出示例 `.zip`

导出文件是空示例文件，目的是保证静态部署环境下交互稳定。

## 移动端策略

移动端不强行压缩 4 列看板，而是采用更适合窄屏的交互：

- 横向滑动查看不同申请阶段。
- 页面提示“左右滑动查看不同申请阶段”。
- 搜索或筛选后自动隐藏无结果阶段。
- 单阶段结果下禁用多余横向滑动，避免滑到空白区域。

这样可以优先保证信息可读性，而不是机械保留桌面端布局。

## 文件结构

- `index.html`
  - 页面结构
  - 看板、表格、日历、弹窗、抽屉和模板

- `styles.css`
  - 视觉样式
  - 响应式布局
  - 移动端横向滑动看板适配

- `script.js`
  - 示例数据
  - 看板 / 表格 / 日历渲染
  - 搜索、筛选、排序
  - 抽屉编辑
  - 本地持久化
  - `.csv`、`.ics` 和示例材料导出

## 实现边界

当前项目是前端静态原型，不包含真实后端能力：

- 不接入真实招聘平台 API。
- 不包含账号系统。
- 不支持多人协作。
- 不支持跨设备同步。
- 材料导出和日历导出使用示例文件。

这些边界可以保证项目在纯静态环境下稳定运行，也方便直接部署和演示。
